# Play Console 데이터 안전(Data Safety) 양식 작성 가이드

**앱 이름:** 또르르 미로
**패키지명:** com.rts.rys.ryy.wayfinding
**최종 정리일:** 2026-07-08

대전 모드(Nearby Connections 기반 로컬 1:1 대전) 출시를 반영한 데이터 안전 양식 작성 내용.
핵심 결론: **"수집·공유하는 데이터 없음"으로 신고 가능.**

---

## 1. 데이터 수집 및 보안 (Data collection and security)

| 질문 | 답변 | 근거 |
|---|---|---|
| 앱이 필수 사용자 데이터 유형을 수집하거나 공유하나요? | **아니요 (No)** | 서버·인터넷·계정 없음. 모든 데이터는 기기 내부에만 저장 |
| (수집 시) 전송 중 데이터가 암호화되나요? | 해당 없음 | 외부로 전송하는 데이터 자체가 없음 |
| 사용자가 데이터 삭제를 요청할 수 있나요? | 해당 없음 / 앱 삭제 시 전부 삭제 | 모든 로컬 데이터는 앱 제거 시 함께 삭제 |

→ 결과적으로 데이터 유형별 상세 표는 **작성 대상이 없습니다.**

## 2. "수집 없음"으로 신고할 수 있는 근거 (심사 소명용)

Google의 "수집(Collected)" 정의 = **기기 밖(개발자 서버/제3자)으로 데이터를 전송**하는 것. 본 앱은 이에 해당하지 않는다.

- **기기 내부 저장 ≠ 수집**: 게임 기록·닉네임·대전 결과는 `SharedPreferences`(기기 내부)에만 저장. 개발자에게 전송되지 않음.
- **대전 모드 기기 간 연결 ≠ 수집/공유**:
  - 인터넷·서버 미경유, 같은 공간의 두 기기만 직접 연결(Nearby Connections, 블루투스/Wi-Fi Direct).
  - 교환 데이터는 **일시적 게임 상태값**(공 좌표·점수·진행도·시작/완료 신호)뿐이며 **매치 종료 시 즉시 폐기**(ephemeral) → Google이 수집에서 제외하는 "일시적 처리" 요건 충족.
  - 개발자나 제3자에게 전달되는 경로가 없어 "공유(Shared)"에도 해당하지 않음.
- **닉네임은 개인정보 아님**: 앱이 준비한 목록에서 선택하는 값(예: "토끼17"), 자유 입력·실명·식별자 불가.

## 3. 함께 갱신/확인할 항목

- **권한 선언**: "근처 기기"(블루투스/Wi-Fi) 권한 사유를 **"로컬 1:1 멀티플레이 연결용 — 위치 추적 아님"**으로 기재. `neverForLocation` 플래그 반영됨.
- **광고 ID**: 사용 안 함 유지 (targetSdk 35, AAID 권한 미선언).
- **대상 연령 / Families 정책**: 아동 대상 앱 → 데이터 안전 답변이 Families 정책과 일치하는지 확인. (채팅 없음·PII 없음·위치 없음이라 리스크 낮음)
- **개인정보처리방침 URL**: 갱신된 `PRIVACY_POLICY.md` 게시 URL을 콘솔에 등록/확인.

## 4. 근처 기기 권한 사용 목적 (콘솔 문의 대비)

대전 모드는 "근처 기기" 권한을 런타임으로 요청하므로, 심사 시 권한 사용 목적을 별도로 물을 수 있다. 답변:

> 동일 공간에 있는 기기 간 로컬 1:1 대전을 연결하기 위한 용도이며, 위치 정보 수집이나 사용자 추적 목적이 아닙니다.

- 요청 권한: `NEARBY_WIFI_DEVICES`, `BLUETOOTH_ADVERTISE`, `BLUETOOTH_CONNECT`, `BLUETOOTH_SCAN` (모두 "근처 기기" 권한 그룹).
- Android 12L(API 32) 이상에서만 제공, 대전 모드 진입 시에만 요청.
- 위치 권한(ACCESS_FINE/COARSE_LOCATION)·저장소 권한은 선언·요청하지 않음.
