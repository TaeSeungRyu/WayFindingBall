# 별자리 만들기 — 종횡비 눌림·이모지 피커 잘림 수정

별자리 만들기에서 (1) 저장한 별자리를 플레이하면 모양이 눌려 보이고, (2) 저장 다이얼로그 "그림 고르기"의 우측 이모지가 잘리던 두 문제를 수정했습니다. 2.0.4 (versionCode 9)에 반영.

---

## ✅ 구현 완료 (2026-07-14 기준)

- **종횡비 눌림 수정** — `ui/ConstellationCreateScreen.kt`의 `StarCanvas`. 만들기 캔버스가 `fillMaxWidth().weight(1f)`라 세로로 긴 직사각형이었고, 별 좌표를 그 직사각형 기준(0..1)으로 저장했다. 그런데 플레이 화면(`ui/ConstellationGameScreen.kt`의 `ConstellationCanvas`)은 `minOf(maxWidth, maxHeight)`로 만든 **정사각형**에 렌더링하므로, 직사각형 좌표를 정사각형에 되매핑하면서 세로가 눌렸다. 만들기 캔버스도 `BoxWithConstraints` + `size(정사각형)`으로 바꿔 플레이와 종횡비를 일치시켰다.
- **이모지 피커 잘림 수정** — 같은 파일의 `EmojiPicker`. 고정 `44.dp` 셀 6개 + 간격이 한 줄에 304dp를 요구해 좁은 다이얼로그에서 우측이 잘렸다. 셀을 `weight(1f).aspectRatio(1f)`로 바꿔 다이얼로그 폭에 맞춰 항상 6개가 들어가도록 했다.

## 왜 이렇게 고쳤나

- 좌표는 그대로 정규(0..1)라 **기존에 저장된 별자리도 재변환 없이** 새 정사각형 기준으로 올바르게 표시된다. 마이그레이션 불필요.
- 만들기·플레이가 동일한 정사각형 렌더링 규칙을 공유하므로, 앞으로도 두 화면의 모양이 항상 일치한다.

## 영향 범위

- 만들기 화면 캔버스가 세로로 긴 영역 안에서 정사각형으로 중앙 정렬된다(위/아래 여백 발생). 플레이 화면과 동일한 배치.
- 데이터 포맷·저장소(`data/CustomConstellationRepository.kt`) 변경 없음.
